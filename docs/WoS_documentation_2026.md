# Web of Science (WoS) Data Extraction — 2026

## Overview

The WoS module extracts and aggregates publication and citation data for Marshall School of Business faculty using the **Web of Science Starter API** (Clarivate). It uses each faculty member's unique WoS ResearchID to query all their publications, then filters to the most recent 5 years and ranks faculty by total citations.

## Setup

### Prerequisites

- Python 3.9+
- Virtual environment created and activated (see README for setup)
- Dependencies installed inside the venv: `pip install -r code/requirements.txt`
- `.env` file with your WoS API key (see below)

### WoS API Key Setup

1. Register at the [Clarivate Developer Portal](https://developer.clarivate.com/)
2. Subscribe to the **Web of Science Starter API**: https://developer.clarivate.com/apis/wos-starter
3. Once approved, copy your API key
4. Add it to `code/.env`:
   ```
   WOS_API_KEY=your_key_here
   ```

## API Details

- **Base URL:** `https://api.clarivate.com/apis/wos-starter/v1/documents`
- **Authentication:** `X-ApiKey` header
- **Query format:** `AI=(researcher_id)` (Author ID search)
- **Database:** `WOK` (Web of Knowledge — broadest coverage)
- **Pagination:** 50 records per page
- **Rate limiting:** 1-second pause between pages
- **Retry logic:** Up to 3 retries with exponential backoff on HTTP errors



## Year Window

The year window is defined in `config.py` and computed dynamically:

```
YEAR_END   = current_year - 1  (last complete year)
YEAR_START = YEAR_END - 4      (5-year window)
```

In 2026: **2021–2025**



## Known Limitations

- Faculty without a WoS ResearchID cannot be queried — they appear in the error log.
- Citation counts in WoS are typically lower than Google Scholar (WoS covers only indexed journals; Google Scholar includes preprints, working papers, etc.).
